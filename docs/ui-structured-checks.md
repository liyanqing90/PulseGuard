# UI 结构化校验构思

目标：让 UI 任务像接口任务一样，优先通过可视化操作完成配置，脚本只作为高级模式保留。

## 推荐交互

1. 发起页面预览
   - 输入 URL 后点击“打开页面”。
   - 系统启动 Playwright 预览页，采集页面标题、URL、可见文本、可点击元素、输入框、按钮、链接和截图。

2. 录制操作步骤
   - 用户在预览页点击、输入、选择、等待。
   - 系统生成结构化步骤，而不是直接生成 Python 脚本。
   - 步骤类型：打开页面、点击元素、输入文本、选择选项、等待元素、等待 URL、等待网络空闲。

3. 点选元素添加校验
   - 用户在预览页点选元素。
   - 右侧显示元素名称、角色、文本、稳定选择器候选、可见状态。
   - 可一键添加校验：元素可见、元素隐藏、文本包含、文本等于、元素数量、属性值、输入框值、按钮可用、URL 包含、标题包含。

4. 调试结果内联展示
   - 不跳新页面，编辑器底部直接展示回放结果。
   - UI 结果优先展示：步骤列表、每步状态、失败步骤、截图、Trace、控制台错误。
   - 失败时自动定位到失败步骤和对应截图。

5. 高级模式
   - 保留 Python 脚本折叠区。
   - 结构化步骤存在时默认不要求脚本。
   - 需要复杂登录、多窗口、业务分支时再使用脚本。

## 数据结构草案

```json
{
  "steps": [
    {"type": "goto", "url": "$entry_url"},
    {"type": "click", "selector": "role=button[name='登录']"},
    {"type": "fill", "selector": "label=手机号", "value": "13800000000"}
  ],
  "assertions": [
    {"type": "element_visible", "selector": "text=首页"},
    {"type": "text_contains", "selector": "[data-testid='user-name']", "expected": "张三"},
    {"type": "url_contains", "expected": "/dashboard"}
  ]
}
```

## 第一阶段实现切片

- 新增 `ui_steps_json` 和 `ui_assertions_json`。
- UI 编辑器增加“打开页面预览”和“添加校验项”。
- 先支持 6 个规则：页面标题包含、URL 包含、文本出现、元素可见、元素隐藏、元素数量。
- 回放结果用表格展示：步骤、操作、目标、状态、耗时、错误。
- 高级脚本继续保留，结构化 UI 校验存在时不要求脚本。
