# PulseGuard 未来发展任务路线

> 当前任务系统基线：本文件用于承接 V0.5 到 V2.0 的长期任务拆解。每个版本只保留与 PulseGuard 本地 / 内网 UI/API 探活定位一致的任务。

## 产品边界与跨版本约束

- 可做内网只读状态页；只展示脱敏后的任务组状态、最近失败和基础指标，不提供公开订阅、SLA 报表或面向公网业务用户的状态页。
- 可做轻量维护公告；只服务内网展示、告警静默提示或运行上下文，不进入确认回执、事件生命周期、incident workflow 或 on-call 系统。
- 不把录制器作为主线；除非后续出现大量多步骤探活复用需求，否则只保留为 V2.0 之后的实验方向。
- 不做复杂 E2E 编排平台。
- 不做完整 incident / on-call 系统。
- 不急于做 SaaS 或复杂多用户 RBAC。
- `setup_script` 是 UI 探活的跨流程前置阶段：扫描、草稿调试、正式运行都必须先执行前置脚本；前置脚本失败时中止本次流程，并输出脱敏后的失败原因。
- UI 和交互必须遵守 `PRODUCT.md`、`DESIGN.md`、`frontend/src/designSystem.ts`、`frontend/src/styles.css`。
- 前端优先使用 Ant Design、成熟第三方组件和现有业务组件，不新增 primitive wrapper。

## V0.5 已交付基线

- [x] UI 扫描支持 `setup_script`：扫描、草稿调试、正式运行统一使用前置脚本准备页面前提。
- [x] 环境变量与密钥占位符：支持 `${BASE_URL}`、`${TOKEN}`，覆盖 API/UI 入口、Headers、Body、结构化断言、浏览器代理、通知 Webhook 和钉钉密钥；变量公开设置、运行记录、调试返回和告警错误默认脱敏。截图与 Trace 属于二进制/浏览器产物，后续归档策略中继续收口访问边界。
- [x] 配置导入导出：任务、设置、通知渠道可迁移，导入前预检，导出默认脱敏；重名任务默认自动改名，显式替换时更新同名任务。
- [x] 失败对比：失败/超时运行对比同任务最近一次成功运行，按可用快照展示 URL、标题、API 状态码和关键断言差异；不扩展为完整视觉回放或调试平台。
- [x] 任务级告警策略：支持按任务或标签覆盖全局冷却、恢复通知和通知渠道；优先级为任务 > 标签 > 全局，任务/标签策略只引用全局通知渠道 ID，不保存 Webhook 或密钥。
- [x] 轻量趋势：基于现有运行记录展示近 24h / 7d 成功率、失败次数和耗时 P50/P95；仅做控制台摘要，不扩展为长期 SLA 报表或高容量历史分析。

## V0.6 规则维护效率

- [x] UI selector 稳定性提示：扫描候选生成规则时保留 selector 类型、稳定性和评分；低稳定性 selector 保存到草稿前弹出确认，并在规则表显示稳定性标签。
- [x] 规则失效检测：提供 UI 规则检测接口，复用前置脚本和页面准备流程；前端可一键检测 selector 规则，并标记正常、未匹配、多匹配、选择器无效等状态，不自动改写规则。
- [x] API 响应预览增强：基于后端 `json_paths` 字段元数据展示响应字段，并可一键生成字段存在、类型、非空、长度基础规则；大响应 body 截断时仍可生成字段规则。
- [x] 任务模板：新建任务时可套用登录页、后台首页、健康接口、列表接口模板；模板只填充草稿字段、前置脚本和结构化校验，不自动保存。
- [x] 批量操作：按标签批量启用、禁用、执行、调整频率；批量运行只执行启用任务，批量变更用命中数量防止列表变化后误操作。
- [x] CLI / CI 入口：新增独立 CLI 模块，可按任务 ID、类型、标签运行已配置探活，输出 JSON，并用 exit code 区分通过、探活失败、无匹配、参数错误和内部异常；不导入 Web `main.py`，不扩展成测试管理平台。

V0.6 交付说明：

1. UI selector 元数据保留与规则失效检测已完成：不改数据库，仅扩展 `assertions_json` 白名单字段和检测接口。
2. 批量操作已完成：统一标签 token 规范，后端校验命中数量，批量运行只执行启用任务。
3. CLI / CI 已完成：独立后端 CLI 模块不导入 Web `main.py`，输出精简 JSON 并用 exit code 表达流水线结果。

## V0.7 克制扩展探活类型

- [x] 被动心跳任务：支持 cron、备份、批处理通过 `POST /api/heartbeats/{key}` 上报健康，API 任务模板可按 freshness 检查最近外部心跳。
- [x] TLS 证书到期检查：通过 API 任务模板和 `ctx.tls_certificate()` 实现，不新增独立任务类型。
- [x] HTTP keyword / redirect / asset 检查：通过 API 任务模板和 `ctx.assert_body_contains()`、`ctx.assert_redirect_occurred()`、`ctx.assert_url_contains()` 实现。
- [x] TCP / DNS / 端口可达检查：通过 API 任务模板和 `ctx.tcp_connect()`、`ctx.dns_resolve()` 实现，V0.7 不引入 `dnspython` 或复杂记录类型。

## V1.0 小团队长期运行

- [x] 操作审计：记录任务、设置、配置导入和批量操作的关键变更，并在运维审计页可查看审计载荷。
- [x] 任务变更历史：任务创建、更新、删除和恢复前保留版本快照，运维审计页可查看并确认恢复。
- [x] 只读访问或轻量访问令牌：提供只读快照令牌，支持 Bearer / Header 访问，设置页可配置或清空令牌，不扩展复杂 RBAC。
- [x] 数据归档策略：删除过期运行记录前按日期、类型和状态保留长期摘要，截图、Trace、Response Body 按独立保留期清理。
- [x] 只读指标出口：提供 JSON 和 Prometheus 格式供外部 Grafana / Prometheus 接入，不演进为托管监控平台。

## V1.2 多 Runner 私有探针

- [x] 主控台负责任务、历史、设置和告警：任务定义、运行历史、告警策略、审计、归档和指标仍由主控台管理。
- [x] Runner 执行面基线：当前本机 Runner 执行探活并写入运行结果，局域网 Runner 先通过注册/心跳进入状态表，不提供远程任务下发或分布式调度。
- [x] Runner 元数据、心跳和状态列表：提供本机 Runner 元数据设置、Runner 心跳注册和 Runner 状态列表，用名称、地址和网络区域识别私有探针。
- [x] 每次运行记录 runner 名称、地址、网络区域和浏览器版本：运行记录、列表、详情和运维页可追溯执行位置与浏览器环境。
- [x] 运行结果区分目标失败和 Runner 环境失败：标准化 `failure_kind`，目标断言/响应/超时失败归为 `target`，Runner 关闭、队列、取消、清理或内部执行异常归为 `runner`。

## V2.0 高级辅助能力

- [x] 内网状态页：作为同控制台内网展示接口，基于脱敏状态页快照展示任务状态、最近异常、运行指标和维护提示，不暴露 error_stack、脚本、Headers、Webhook、环境变量、响应体或 Runner 拓扑。
- [x] 轻量维护公告：允许配置当前维护窗口或公告文本，用于状态页展示，不做订阅、确认回执或 incident 生命周期。
- [x] 失败摘要：提供本地脱敏摘要和建议动作，基于运行状态、错误摘要、Runner 归因和耗时信号生成解释，不自动判定页面正常，不默认外传截图、Trace 或响应体。
- [x] 配置导入导出基线：配置 JSON 可提交到 Git 管理，导出默认脱敏，导入前提供预检；不做声明式 GitOps、diff/apply、Git 服务集成、审批流或远程同步。

## V2.1 后续增强方向

- [ ] AI 辅助生成规则：从页面扫描结果或 API 响应生成建议断言，用户显式确认后保存，不自动改写线上规则；若接入外部模型，必须先展示发送内容并默认脱敏。
- [ ] Playwright 用例导入：允许复用已有脚本生成 PulseGuard 任务草稿，但不扩展成完整测试管理平台、报告矩阵或复杂 E2E 编排。

## V2.x 远期观察项

- 录制器实验：当前不进入主线，也不是当前版本承诺项；除非出现大量多步骤探活复用需求，否则优先继续用模板、前置脚本和结构化规则。若后续评估录制，录制前也必须先执行 `setup_script`，前置失败时中止并返回脱敏原因。

## 验收原则

- 每项任务必须有后端单元测试或前端构建验证。
- 涉及 UI 的任务必须保持 Ant Design 控件和现有设计系统约束。
- 涉及脚本、密钥、通知、导入导出的任务必须明确脱敏和失败边界。
- 不通过临时 fallback 隐藏未解决缺陷。
